# Windows DFIR Evidence Ecosystem

Windows-first DFIR triage and reporting toolkit for incident response, evidence preservation, and analyst-friendly reporting.

This project supports two operating modes:

1. **Standalone mode**  
   Run a portable PowerShell collector on a Windows endpoint and build a local forensic report.

2. **Velociraptor mode**  
   Use a custom Velociraptor artifact to run the same collector at scale, including offline collector workflows.

---

## Current capabilities

### 1) Richer Windows forensic artefacts
The collector now supports broader triage coverage, including:

- Windows event logs
- processes, services, drivers
- scheduled tasks
- Run / RunOnce persistence
- Defender status and detections
- network listeners and connections
- Prefetch
- LNK files
- Recent files
- Jump Lists
- Amcache
- SRUM
- USB history
- optional browser artefacts
- optional placeholders for memory and YARA workflows

### 2) Consolidated forensic timeline
The reporting pipeline now generates:

- `08_timeline/timeline.csv`
- `08_timeline/timeline.json`

The timeline consolidates:

- collection metadata
- Defender detections
- scheduled tasks
- services
- Run keys
- TCP listeners
- recent file activity
- findings generated by the detection engine

### 3) Improved findings engine
The findings engine now supports:

- severity scoring
- category tagging
- confidence tagging
- encoded PowerShell detection
- suspicious service path detection
- LOLBin detection
- suspicious listener port detection
- WMI subscription review
- Defender posture checks
- correlation finding for probable compromise clusters

### 4) Better Velociraptor integration
The custom artifact now supports parameters for:

- `CaseId`
- `Profile`
- `MaxDays`
- `IncludeBrowserArtefacts`
- `IncludeMemory`
- `IncludeYara`
- `ZipSharePackage`

### 5) Reduced external analysis package
The project now generates a reduced package under:

```text
99_share_with_internal_or_controlled_ai
```

This package includes:

- `summary.json`
- `findings.json`
- `timeline.csv`
- `timeline.json`

It can also be zipped automatically into:

```text
<CASEID>-share-package.zip
```

## Quick start

### Reporter setup on macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
python -m pip install pytest
python -m playwright install chromium
```

### Validate the reporting pipeline

```bash
python -m pytest
python -m reporter.report_builder   --case-path sample_case/CASE-DEMO-HOST01   --zip-share-package
```

### Render PDF and screenshots

```bash
python -m reporter.report_builder   --case-path sample_case/CASE-DEMO-HOST01   --render-pdf   --take-screenshots   --zip-share-package
```

### Open the demo report on macOS

```bash
open sample_case/CASE-DEMO-HOST01/11_reports/report.html
```

## Manual evidence collection on Windows

Open an elevated PowerShell console:

```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
.\collector\windows\Invoke-DFIREvidenceCollector.ps1 `
  -CaseId CASE-2026-0001-HOST01 `
  -OutputRoot C:\DFIR `
  -Profile Incident `
  -IncludeBrowserArtefacts `
  -ZipSharePackage `
  -MaxDays 14
```

Optional switches:

- `-IncludeBrowserArtefacts`
- `-IncludeMemory`
- `-IncludeYara`
- `-ZipSharePackage`
- `-NoZip`

Expected output:

```text
C:\DFIR\CASE-2026-0001-HOST01\
C:\DFIR\CASE-2026-0001-HOST01.zip
```

### Build a report from a real case

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python -m pip install pytest
python -m playwright install chromium

python -m reporter.report_builder `
  --case-path C:\DFIR\CASE-2026-0001-HOST01 `
  --render-pdf `
  --take-screenshots `
  --zip-share-package
```

Outputs are written under:

```text
C:\DFIR\CASE-2026-0001-HOST01\11_reports\
C:\DFIR\CASE-2026-0001-HOST01\08_timeline\
C:\DFIR\CASE-2026-0001-HOST01\99_share_with_internal_or_controlled_ai\
```

## Velociraptor installation on macOS (Apple Silicon)

Create a working directory:

```bash
mkdir -p ~/tools/velociraptor
cd ~/tools/velociraptor
```

Download the binary:

```bash
curl -L -o velociraptor https://github.com/Velocidex/velociraptor/releases/download/v0.76/velociraptor-v0.76.1-darwin-arm64
chmod +x velociraptor
```

Confirm the binary:

```bash
./velociraptor version
```

Start the local GUI:

```bash
./velociraptor gui
```

## Velociraptor artifact integration

The custom artifact is stored here:

```text
velociraptor/artifacts/Custom.Windows.DFIR.EvidenceCollector.yaml
```

### Important

Update the `url:` value in the artifact so it points to the real GitHub raw path of your collector script.

Expected pattern:

```text
https://raw.githubusercontent.com/<github-user>/<repo-name>/main/collector/windows/Invoke-DFIREvidenceCollector.ps1
```

Example:

```text
https://raw.githubusercontent.com/nepttunus/windows-dfir-ecosystem/main/collector/windows/Invoke-DFIREvidenceCollector.ps1
```

### Import the artifact into Velociraptor

1. Open the Velociraptor GUI
2. Go to **View Artifacts**
3. Click **New Artifact**
4. Paste the full YAML content of:
   `velociraptor/artifacts/Custom.Windows.DFIR.EvidenceCollector.yaml`
5. Save the artifact

### Suggested first lab run

- `CaseId = CASE-LAB-001`
- `Profile = Incident`
- `MaxDays = 14`
- `IncludeBrowserArtefacts = false`
- `IncludeMemory = false`
- `IncludeYara = false`
- `ZipSharePackage = true`

## Operational notes

- Run as local administrator whenever possible
- This tool is intended for forensic triage and reporting, not full dead-box acquisition
- Some artefact coverage varies by Windows version and endpoint controls
- Browser collection may depend on file locks and permissions
- Memory and YARA are optional workflows and currently placeholder-ready
- Rebuild offline collectors after Velociraptor server upgrades or artifact changes

---

## Limitations

- Does not replace full forensic imaging
- Does not replace memory forensics tooling
- Does not replace analyst-led forensic interpretation
- Intended for fast triage, structured reporting, and evidence packaging
